## Текстовое описание юзкейса

---

## Рамки
Приложение информационной системы деканата образовательного учреждения.

## Уровень  
Задача, определенная пользователем.

## Основной исполнитель  
Сотрудник деканата.

## Заинтересованные лица и их требования
- Сотрудник деканата. Хочет быстро увеличить номер курса группы без дополнительных проверок и подтверждений.

## Предусловия
1. Сотрудник деканата идентифицирован и аутентифицирован в системе.
2. Группа существует в системе.

## Результаты (Постусловия)
- Номер курса группы увеличен на единицу.
- Документ об изменении номера курса зарегистрирован в системе.

## Основной успешный сценарий
1. Сотрудник деканата выбирает группу в интерфейсе системы.
2. Сотрудник нажимает кнопку "Перевести на следующий курс".
3. Система увеличивает значение номера курса группы на 1.
4. Система регистрирует документ об изменении номера курса.
5. Процесс завершен.

## Расширения (альтернативные потоки)
а. Ошибка при сохранении данных
1. Система не может сохранить изменения в базе данных.
2. Изменение номера курса откатывается.
3. Сотрудник получает сообщение об ошибке.

## Специальные требования
- Интерфейс должен содержать одну кнопку для выполнения операции.
- Операция должна выполняться быстро (менее 3 секунд).

## Список технологий и типов данных
- Простое обновление поля в базе данных.
- Регистрация документа в системе электронного документооборота.

## Частота использования
1-2 раза в год для каждой группы.

## Открытые вопросы
- Нет.
