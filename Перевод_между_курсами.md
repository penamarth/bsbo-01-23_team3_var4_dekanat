## Рамки
Приложение информационной системы деканата образовательного учреждения.

## Уровень  
Задача, определенная пользователем.

## Основной исполнитель  
Сотрудник деканата.

## Заинтересованные лица и их требования
- **Сотрудник деканата.** Хочет корректно и оперативно оформить перевод студента на следующий курс, минимизировать ручной ввод данных, избежать ошибок при оформлении документов.
- **Студент.** Хочет продолжить обучение на следующем курсе при успешном завершении текущего.
- **Учебно-методический отдел.** Хочет своевременно получить информацию о переводе для формирования списков групп следующего курса и корректировки учебной нагрузки.
- **Система электронного документооборота.** Хочет автоматически получать данные о переводе для регистрации приказа.

## Предусловия
1. Сотрудник деканата идентифицирован и аутентифицирован в системе.
2. Студент числится в базе данных образовательного учреждения, зарегистрирован в текущей группе и успешно завершил учебный год (сдал все экзамены, не имеет академической задолженности).
3. Существует учебная группа следующего курса, соответствующая направлению обучения студента.

## Результаты (Постусловия)
- Студент переведён в группу следующего курса.
- Данные о переводе сохранены в системе.
- Приказ о переводе сформирован и зарегистрирован в системе электронного документооборота.
- Учебно-методический отдел уведомлён о переводе.
- Бухгалтерия получает данные для пересчёта стипендии (при наличии).
- Студент уведомлён о переводе.

## Основной успешный сценарий
1. Сотрудник деканата выбирает операцию «Перевод на следующий курс».
2. Система отображает форму для поиска студента.
3. Сотрудник вводит идентификатор студента (ФИО или номер зачётной книжки).
4. Система отображает профиль студента, текущую группу и успеваемость.
5. Система автоматически проверяет отсутствие академической задолженности.
6. Сотрудник выбирает целевую группу следующего курса (система может предложить её автоматически).
7. Сотрудник отправляет приказ на перевод на согласование декану.
8. Декан подтверждает перевод студента.
9. Система регистрирует перевод:
   - Обновляет курс и группу студента.
   - Формирует приказ о переводе.
10. Система направляет данные в систему электронного документооборота.
11. Система уведомляет учебно-методический отдел о переводе.
12. Система формирует уведомление для студента о переводе на следующий курс.
13. Процесс перевода завершён.

## Расширения (альтернативные потоки)
**а. Декан отклоняет перевод**
1. Система регистрирует отказ.
2. Сотрудник деканата и студент получают уведомление об отклонении перевода.
3. Студент остаётся в текущей группе.

**б. Наличие академической задолженности**
1. При проверке успеваемости система обнаруживает несданные экзамены.
2. Система уведомляет сотрудника деканата о невозможности перевода.
3. Процесс приостанавливается до ликвидации задолженности.

**в. Отсутствие доступной группы следующего курса**
1. Система не находит подходящей группы для перевода.
2. Сотруднику деканата предлагается создать новую группу или выбрать альтернативный вариант.
3. Процесс приостанавливается до решения вопроса с группой.

## Специальные требования
- Интерфейс системы должен быть интуитивно понятным и доступным для пользователей с разным уровнем компьютерной грамотности.
- Система должна обеспечивать быстрое выполнение операций (отклик менее 5 секунд).
- Данные должны быть защищены в соответствии с федеральными законами о защите персональных данных.
- Интеграция с системой электронного документооборота для автоматической регистрации приказов.
- Автоматическая проверка успеваемости студента перед переводом.
- Возможность массового перевода группы студентов на следующий курс.

## Список технологий и типов данных
- Использование электронной подписи для подписания приказов.
- Поддержка различных форматов документов (PDF, DOCX).
- Автоматическая генерация уведомлений по электронной почте и в личный кабинет студента.
- Интеграция с системой успеваемости для проверки академического статуса студента.

## Частота использования
Регулярно, в конце каждого учебного года (1–2 раза в год).

## Открытые вопросы
- Процедура перевода студентов с академической задолженностью после пересдачи.
- Особенности перевода студентов, обучающихся на платной основе.
- Автоматическое формирование приказов о переводе для целой группы студентов.
- Интеграция с системой бухгалтерского учёта для перерасчёта стипендий.
