
Рамки
Модуль "Внесение данных об успеваемости" в составе информационной системы образовательного учреждения.

Уровень
Задача, определенная пользователем.

Основной исполнитель
Преподаватель.

Заинтересованные лица и их требования
-   Преподаватель. Хочет быстро и удобно выставлять и просматривать оценки, видеть историю изменений, закрывать аттестации по своим дисциплинам.
-   Сотрудник учебного отдела. Требует своевременного и корректного внесения оценок для формирования официальной отчетности.
-   Декан/Зав. кафедрой. Требует достоверных данных об успеваемости для анализа и принятия управленческих решений.
-   Студент. Хочет оперативно видеть свои оценки и средний балл в личном кабинете.
-   Система расчета стипендий. Требует актуальных данных об оценках для автоматического начисления стипендии.

Предусловия
1.  Преподаватель идентифицирован и аутентифицирован в системе.
2.  Преподаватель имеет в текущем семестре учебную нагрузку (закрепленные дисциплины и группы).
3.  В системе созданы элементы учебного плана: дисциплины, группы, формы контроля.

Результаты (Постусловия)
1.  Оценка внесена в электронный журнал и привязана к студенту, дисциплине и преподавателю.
2.  Рассчитан и обновлен текущий средний балл студента по дисциплине.
3.  Изменения зафиксированы в истории с указанием автора и времени.
4.  Данные об оценке доступны для просмотра уполномоченным лицам (студенту, учебному отделу).

Основной успешный сценарий
1.  Преподаватель выбирает в меню раздел "Мои учебные курсы".
2.  Система отображает список дисциплин и групп, закрепленных за преподавателем в текущем семестре.
3.  Преподаватель выбирает нужную дисциплину и группу.
4.  Система отображает интерфейс журнала успеваемости: список студентов, виды работ и оценок (колонки: "Посещение", "Практическая 1", "Экзамен" и т.д.).
5.  Преподаватель находит студента в списке и вносит оценку в ячейку, соответствующую виду работы.
6.  Система проверяет оценку на соответствие допустимой шкале (например, 5-балльной).
7.  Преподаватель подтверждает ввод.
8.  Система сохраняет оценку, автоматически пересчитывает и отображает текущий средний балл студента.
9.  Преподаватель повторяет шаги 5-8 для других студентов или видов работ.

Расширения (альтернативные потоки)
а. Неверный формат или значение оценки.
    1. Система выводит сообщение: "Недопустимое значение. Используйте шкалу: 2, 3, 4, 5".
    2. Преподаватель вводит корректное значение.
b. Студент отсутствовал.
    1. Преподаватель ставит отметку "Н/Я" (Не явился).
    2. Система фиксирует статус и выделяет запись цветом, указывая на наличие задолженности.
c. Ошибка выбора элемента (дисциплины, группы, вида работы).
    1. Преподаватель использует кнопку "Отмена" или "Назад".
    2. Система возвращает Преподавателя на предыдущий шаг без сохранения изменений.

Специальные требования
-   Интерфейс внесения оценок должен быть максимально простым и требовать минимального количества кликов.
-   Обязательное ведение журнала аудита (истории изменений): кто, когда, какую оценку и на какую изменил.
-   Наличие предупреждения при попытке поставить итоговую оценку при наличии незакрытых задолженностей у студента.
-   Автосохранение вводимых данных каждые 2-3 минуты для предотвращения потери при сбое.

Список технологий и типов данных
-   Веб-интерфейс с адаптивным дизайном (для работы с планшета).
-   Валидация данных на стороне клиента (браузера) и сервера.
-   REST API для предоставления данных о оценках другим системам (по запросу).

Частота использования
Ежедневно во время сессии, несколько раз в неделю в течение семестра.

Открытые вопросы
1.  Может ли Преподаватель исправить уже выставленную итоговую оценку после закрытия ведомости? Каков для этого процесс?
2.  Есть ли необходимость в массовом внесении оценок за один вид работы (например, за тест для всей группы сразу)?
