## Рамки
Приложение информационной системы деканата образовательного учреждения.

## Уровень  
Задача, определенная пользователем.

## Основной исполнитель  
Сотрудник деканата.

## Заинтересованные лица и их требования  
- Сотрудник деканата: Хочет правильно и быстро оформить отчисление, минимизировать рутинные операции, избежать ошибок в документации
- Декан: Хочет контролировать процесс отчисления, получать уведомления о критичных случаях
- Студент: Хочет получить официальное подтверждение отчисления и справку об обучении
- Учебно-методический отдел: Хочет своевременно получать данные об отчислении для корректировки учебных групп и отчетности
- Бухгалтерия: Хочет получать информацию об отчислении для прекращения начислений стипендий
- Государственные образовательные службы: Требуют соблюдения законодательства при отчислении и своевременной передачи данных.

## Предусловия  
1. Сотрудник деканата выполнил вход в систему под своей учетной записью.
2. Студент числится в базе данных информационной системы.
3. Имеются документальные основания для отчисления (академическая задолженность, личное заявление, выговоры и т.д.)

## Результаты (Постусловия)
- У студента установлен статус "отчислен"
- Студент удален из состава своей учебной группы
- Все академические записи и успеваемость студента сохранены в архиве
- Сформирован и зарегистрирован приказ об отчислении
- Учебно-методический отдел получил уведомление об отчислении студента
- Бухгалтерия получила уведомление о прекращении выплат
  
## Основной успешный сценарий
1. Сотрудник деканата выбирает в интерфейсе системы операцию "Отчисление студентов"
2. Система отображает форму для поиска студента
3. Сотрудник вводит идентификатор студента (номер студенческого билета или ФИО)
4. Система находит и отображает профиль студента и его текущую группу
5. Сотрудник выбирает из списка основание для отчисления
6. Система автоматически проверяет историю успеваемости и задолженности студента
7. Сотрудник подтверждает отчисление
8. Система выполняет следующие действия:
   - Устанавливает студенту статус "отчислен"
   - Исключает студента из состава учебной группы
   - Регистрирует в системе электронный приказ об отчислении
9. Система направляет приказ на электронную подпись декану
10. После того как декан подписал приказ, система:
    - Направляет сведения об отчислении в учебно-методический отдел
    - Сообщает в бухгалтерию о необходимости прекратить выплаты
11. Система формирует уведомление для студента об отчислении
12. Процесс отчисления завершен
    
## Расширения (альтернативные потоки)
**а. Декан отклоняет отчисление**
1. Система регистрирует факт отказа в подписании приказа
2. Система автоматически возвращает студенту прежний статус и восстанавливает его в группе
3. Сотрудник деканата получает уведомление о том, что декан отклонил отчисление

**б. Отсутствуют необходимые документы**
1. Система проверяет наличие всех обязательных документов, прикрепленных к делу студента
2. Если документов нет, процесс отчисления приостанавливается
3. Система уведомляет сотрудника деканата о необходимости запросить недостающие документы

**в. Студент имеет академическую задолженность**
1. Система при проверке успеваемости обнаруживает у студента академическую задолженность
2. Система автоматически предлагает "академическую неуспеваемость" в качестве основания для отчисления
3. Процесс продолжается с выполнением дополнительных проверок успеваемости

**г. Ошибка в данных студента**
1. При поиске и отображении профиля студента система обнаруживает несоответствие или ошибку в данных
2. Процесс отчисления приостанавливается
3. Система требует от сотрудника исправить данные в карточке студента перед продолжением

## Специальные требования
- Интерфейс системы должен быть интуитивно понятным и доступным для пользователей с разным уровнем компьютерной грамотности
- Система должна обеспечивать быстрое выполнение операций (отклик менее 5 секунд)
- Данные должны быть защищены в соответствии с федеральными законами о защите персональных данных
- Обеспечена интеграция с системой электронного документооборота для работы с приказами
- Обеспечена синхронизация данных об учебных группах после исключения студента
- Все операции по отчислению выполняются как единая транзакция (либо все шаги выполняются, либо ни одного)
- Автоматическая проверка академической истории студента выполняется при выборе основания для отчисления

## Список технологий и типов данных
- Использование баз данных для хранения сведений о студентах и учебных группах
- Работа с базой данных для сохранения всех изменений
- Использование системы электронной подписи документов
- Интеграция с внешними системами (УМО, Бухгалтерия, Электронный документооборот) через стандартизированные интерфейсы

## Частота использования
Несколько раз в семестр, в зависимости от количества отчислений.

## Открытые вопросы
- Процедура восстановления студентов после отчисления
- Процедура отчисления по собственному желанию
- Автоматическое уведомление студентов через личный кабинет
- Изучить приказы для обеспечения конфиденцииальности персональных данных
